**Create `src/components/frontend/InkTransition.tsx`** — a client component that renders a full-viewport ink-bleed overlay on route navigation.

```tsx
'use client'
import { useEffect, useState } from 'react'
import { useReducedMotion, motion } from 'motion/react'

export function InkTransition() {
  const prefersReducedMotion = useReducedMotion()
  const [shouldAnimate, setShouldAnimate] = useState(false)

  useEffect(() => {
    // Guard 1: If intro hasn't been seen, skip (prevents conflict with IntroAnimation)
    if (!localStorage.getItem('intro-seen')) return

    // Guard 2: If this is the first mount in this session (initial page load), skip
    // Only animate on subsequent navigations
    if (sessionStorage.getItem('navigated')) {
      setShouldAnimate(true)
    } else {
      sessionStorage.setItem('navigated', '1')
    }
  }, [])

  if (!shouldAnimate || prefersReducedMotion) return null

  return (
    <motion.div
      className="fixed inset-0 z-50 pointer-events-none"
      initial={{ opacity: 1 }}
      animate={{ opacity: 0 }}
      transition={{ duration: 0.3, delay: 0.5, ease: 'easeOut' }}
      aria-hidden="true"
    >
      <svg
        viewBox="0 0 100 100"
        preserveAspectRatio="none"
        className="w-full h-full"
      >
        <defs>
          <filter id="ink-bleed-transition" x="-10%" y="-10%" width="120%" height="120%"
            colorInterpolationFilters="sRGB">
            <feTurbulence type="fractalNoise" baseFrequency="0.04 0.08" numOctaves={3} seed={5} result="noise" />
            <feDisplacementMap in="SourceGraphic" in2="noise" scale={6}
              xChannelSelector="R" yChannelSelector="G" result="displaced" />
            <feGaussianBlur in="displaced" stdDeviation={1.2} />
          </filter>
          <clipPath id="ink-clip">
            <motion.rect
              x="0" y="0" width="100" height="100"
              initial={{ scaleX: 0 }}
              animate={{ scaleX: 1 }}
              transition={{ duration: 0.5, ease: [0.4, 0, 0.2, 1] }}
              style={{ transformOrigin: 'left center' }}
            />
          </clipPath>
        </defs>
        <rect
          x="0" y="0" width="100" height="100"
          fill="var(--color-bg)"
          filter="url(#ink-bleed-transition)"
          clipPath="url(#ink-clip)"
        />
      </svg>
    </motion.div>
  )
}
```

Key design decisions:
1. **Double guard:** `localStorage 'intro-seen'` prevents conflict with IntroAnimation on first-ever visit. `sessionStorage 'navigated'` prevents firing on initial page load (template.tsx remounts on first load too, not just navigation).
2. **SVG with feTurbulence filter:** Reuses the same displacement filter proven in IntroAnimation for visual consistency. The `clipPath` with `scaleX` sweep creates the ink-bleed-from-left effect. The `feDisplacementMap` + `feGaussianBlur` feathers the edge organically.
3. **Timing:** The clip-path reveal takes 0.5s, then the entire overlay fades out over 0.3s (delayed 0.5s). Total animation: ~0.8s. Content is never blocked because `pointer-events: none` is on the wrapper.
4. **z-50:** Above NavBar (z-40) during animation, but pointer-events: none means NavBar is always clickable.
5. **useReducedMotion:** Returns null entirely if user prefers reduced motion.

**If the SVG clipPath + filter produces a crisp rectangular edge instead of an organic bleed:** Fall back to the simpler CSS clip-path approach described in the research. Replace the SVG with:

```tsx
<motion.div
  className="fixed inset-0 z-50 pointer-events-none bg-[var(--color-bg)]"
  style={{ filter: 'url(#ink-bleed-transition)' }}
  initial={{ clipPath: 'inset(0 100% 0 0)' }}
  animate={{ clipPath: 'inset(0 0% 0 0)', opacity: [1, 1, 0] }}
  transition={{ duration: 0.6, times: [0, 0.65, 1], ease: 'easeInOut' }}
  aria-hidden="true"
/>
```

Use whichever approach produces the more organic ink-bleed edge. The SVG approach is preferred but the CSS fallback is acceptable.

**Create `src/app/(frontend)/template.tsx`** — this file remounts on every navigation (Next.js gives it a new key), triggering InkTransition's entrance animation.

```tsx
'use client'
import { InkTransition } from '@/components/frontend/InkTransition'

export default function Template({ children }: { children: React.ReactNode }) {
  return (
    <>
      <InkTransition />
      {children}
    </>
  )
}
```

This is intentionally minimal. Key points:
1. **`'use client'`** is required because InkTransition is a client component.
2. **`template.tsx` NOT `layout.tsx`** — layouts persist across routes (no remount). Templates receive a new unique key on every navigation and fully remount, which triggers InkTransition's `initial`/`animate` props correctly every time.
3. **No `AnimatePresence`** — entrance-only pattern. Exit animations silently break in App Router production (Next.js #42658, #59349). Not needed here since we only animate the incoming page.
4. **InkTransition renders BEFORE children** — the overlay sits on top of content via fixed positioning but children still render underneath immediately (no content delay).
5. The template wraps content that is INSIDE the layout's `<main>` — the rendering order is: `layout.tsx` > `template.tsx` > `page.tsx`. NavBar and MotionProvider (in layout) persist; template + page remount.
