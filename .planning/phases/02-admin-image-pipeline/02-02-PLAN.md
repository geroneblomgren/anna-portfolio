1. **Create `src/components/admin/QRCodeView.tsx`** — a React Server Component that:

   a. Imports `QRCode from 'qrcode'` and `sharp` for composition
   b. Gets Payload instance via `getPayload({ config })` and reads `site-settings` global for `qrUrl`
   c. Falls back to `'https://annadesign.com'` if qrUrl is empty
   d. Generates QR code PNG buffer:
      ```typescript
      const qrBuffer = await QRCode.toBuffer(url, {
        width: 1024,
        margin: 2,
        color: { dark: '#e0e0e0', light: '#0a0a0a' },
        errorCorrectionLevel: 'H',
        type: 'png',
      })
      ```
   e. Creates branded "Dark Arts by Anna" wordmark using Sharp SVG overlay composite:
      - Create an SVG string with the text "DARK ARTS BY ANNA" in a clean sans-serif font, #e0e0e0 color on transparent background
      - The SVG should be approximately 1024px wide × 80px tall, centered text
      - Use `sharp(qrBuffer).extend({ bottom: 100, background: '#0a0a0a' }).composite([{ input: Buffer.from(svgString), gravity: 'south' }]).png().toBuffer()`
      - This adds a branded footer below the QR code
   f. Converts final buffer to base64 data URL
   g. Renders a Payload admin-styled page with:
      - Heading "QR Code"
      - The QR code image displayed (centered, max-width ~512px for preview)
      - Text showing which URL is encoded
      - A download button/link using `<a href={dataURL} download="dark-arts-by-anna-qr.png">`
      - A note: "Update the QR URL in Site Settings"

   Use Payload's Gutter and SetStepNav components if available, otherwise plain div with padding matching admin panel styles. The component path format for Payload 3.x custom views is important — export must be a named export matching the hash in the config path.

         QR Code
       </Link>
     )
   }
   ```
   Note: This is a client component because Payload's afterNavLinks renders in the client sidebar.

   ```

   - Style: yellow/amber text, small font, below the checkbox

   Alternative simpler approach if Payload's admin component API makes fetch cumbersome: Use a `validate` function on the featured field that checks count server-side and returns a warning string (not an error — use `admin.description` with a dynamic component instead).
