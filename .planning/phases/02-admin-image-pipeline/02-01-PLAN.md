  <name>Task 2: Upgrade Media collection with WebP image processing, blur placeholders, and wire Vercel Blob storage</name>
  <files>src/collections/Media.ts, src/payload.config.ts</files>
  <action>
1. **Upgrade `src/collections/Media.ts`** with the following changes:

   a. Add `imageSizes` with three WebP variants (per RESEARCH.md recommendation for original retention + three variants):
      - `gallery`: width 1200, WebP quality 75, withoutEnlargement true — for gallery display (≤300KB target)
      - `lightbox`: width 2400, WebP quality 90, withoutEnlargement true — high-quality for Phase 3 lightbox
      - `thumb`: width 400, WebP quality 70, withoutEnlargement true — admin grid preview

   b. Add `formatOptions` to each imageSize:
      ```
      formatOptions: { format: 'webp', options: { quality: NN } }
      ```

   c. Add `blurDataURL` text field to the collection fields (stores base64 data URI).

   d. Restrict `mimeTypes` to specific accepted formats: `['image/jpeg', 'image/png', 'image/tiff', 'image/webp']` (per CONTEXT.md locked decision).

   e. Set `filesRequiredOnCreate: false` (blur hook runs after save — field starts empty).

   f. Add `afterChange` hook for blur placeholder generation. CRITICAL TIMING: Use `setTimeout(fn, 100)` wrapper because calling `req.payload.update()` on the same document inside `afterChange` races with the outer save's SQLite transaction commit (see RESEARCH.md Pitfall 1).

   The afterChange hook should:
   - Only run on `create` or when `update` has a new file (`operation === 'create' || (operation === 'update' && req.file)`)
   - Import sharp at the top of the file
   - Read the thumb buffer from `req.payloadUploadSizes?.thumb` if available, or fall back to reading the uploaded file via `req.file?.data`
   - Run `sharp(buffer).resize(10, 10, { fit: 'inside' }).blur(1).png({ compressionLevel: 9 }).toBuffer()`
   - Store as `data:image/png;base64,${blurBuffer.toString('base64')}`
   - Call `req.payload.update({ collection: 'media', id: doc.id, data: { blurDataURL }, overrideAccess: true })` inside the setTimeout
   - Wrap in try/catch, log errors via `req.payload.logger.error()`

   g. Add a **post-processing size check** in the same afterChange hook: after upload, if the gallery WebP variant exceeds 300KB, re-process at reduced quality (step down 75 → 65 → 55) using Sharp directly and update the stored file. This is Claude's discretion per CONTEXT.md. Implementation: read the gallery size from `doc.sizes?.gallery?.filesize`; if over 307200 bytes, log a warning. Full re-compression feedback loop is optional — the warning is the minimum; if straightforward to implement within Payload's storage abstraction, do it.

   h. Set `upload.filenameCompoundIndex` or max upload size. The `adminThumbnail` should use the 'thumb' size. Set upload config:
      ```
      upload: {
        mimeTypes: ['image/jpeg', 'image/png', 'image/tiff', 'image/webp'],
        adminThumbnail: 'thumb',
        imageSizes: [ ... ],  // as above
      }
      ```

2. **Update `src/payload.config.ts`**:

   a. Import and register ArtPieces collection:


   b. Import and configure Vercel Blob storage plugin:
      ```typescript
      import { vercelBlobStorage } from '@payloadcms/storage-vercel-blob'
      ```
      Add to plugins array:
      ```typescript
      plugins: [
        vercelBlobStorage({
          collections: { media: true },
          clientUploads: true,   // REQUIRED: bypass Vercel 4.5MB serverless body limit (Anna uploads up to 20MB)

      ],
      ```
      IMPORTANT: Do NOT use `prefix` option — known bug with clientUploads+prefix in Payload 3.x (RESEARCH.md Pitfall, GitHub Issue #12541).

   c. Remove `staticDir: 'public/media'` from Media collection — Vercel Blob replaces local storage. The storage adapter handles the directory concern.

If not, wrap the plugin in a conditional: `...(process.env.BLOB_READ_WRITE_TOKEN ? [vercelBlobStorage({...})] : [])`.
