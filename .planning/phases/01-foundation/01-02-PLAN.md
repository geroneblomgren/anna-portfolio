<task type="auto">
  <name>Task 1: Deploy to Vercel and configure environment variables</name>
  <files>vercel.json</files>
  <action>
1. Initialize git repo if not already done: `git init`, create initial commit with all project files.

2. Create a GitHub repository for the project (use `gh repo create anna-portfolio --public --source=. --push` or guide through the process).

3. Install Vercel CLI if not present: `npm i -g vercel`.

4. Run `vercel` to link the project to the user's existing Vercel account. Follow prompts to create a new project.

5. Set environment variables on Vercel (these must come from the user's Turso setup):
   ```bash
   vercel env add TURSO_DATABASE_URL production
   vercel env add TURSO_AUTH_TOKEN production
   vercel env add PAYLOAD_SECRET production
   ```
   For PAYLOAD_SECRET, generate a new production value: `openssl rand -base64 32`. Do NOT reuse the local dev value.

6. Deploy: `vercel --prod`

7. Verify the deployment:
   - The build log shows zero errors
   - The production URL returns 200
   - /admin on the production URL loads (may show first-user setup screen — that's correct)

8. If the build fails due to TypeScript errors or missing dependencies, fix them and redeploy.

IMPORTANT: The user's Vercel account already exists (per CONTEXT.md). The Turso credentials must be ready from the user_setup in Plan 01. If they aren't available yet, this task will create a checkpoint.
  </action>

    <automated>vercel ls 2>&amp;1 | head -10</automated>
  </verify>
