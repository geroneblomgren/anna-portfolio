```tsx
<motion.div
  key={piece.id}
  className="gallery-card relative group cursor-pointer overflow-hidden rounded-sm"
  initial={{ opacity: 0, y: 24 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true, margin: '-60px' }}
  transition={{ duration: 0.55, ease: 'easeOut', delay: idx * 0.07 }}
  onClick={() => { setLightboxIndex(idx); setLightboxOpen(true) }}

```

Key details:
- Add `gallery-card` to the className (needed for Task 2 shadow pseudo-element).
- Use `viewport={{ once: true }}` — this is CRITICAL for PERF-03. Without it, every card retains an active Intersection Observer and GPU layer.
- Use `margin: '-60px'` so cards start animating slightly before entering viewport (feels more natural).
- Use per-card `delay: idx * 0.07` — NOT parent staggerChildren. Masonry renders 3 column containers, so parent stagger would stagger columns, not individual cards.
- Cap the delay for large galleries: `delay: Math.min(idx, 12) * 0.07` to prevent last cards having a 1.4s+ delay.

3. Close the motion.div at the matching closing tag (was line 89, the `</div>`). Change it to `</motion.div>`.

  <files>src/components/frontend/GalleryGrid.tsx, src/app/globals.css</files>
  <action>
1. In GalleryGrid.tsx, add `whileHover` to the motion.div created in Task 1:

```tsx
whileHover={{ scale: 1.025 }}
```

Add this prop to the motion.div. Motion's whileHover naturally only fires on hover-capable devices in the browser, but the CSS shadow pseudo needs explicit gating (see step 2).

The existing gradient overlay div (line 79) already uses `opacity-0 group-hover:opacity-100 transition-opacity` via Tailwind's `group-hover:` — this handles the gradient depth lift. However, Tailwind's `group-hover:` needs CSS gating too. Wrap the existing hover overlay's visibility in a media query by adding this to globals.css (step 2).

2. In globals.css, add the following rules AFTER the tag-filter-scroll rules (line 25) and BEFORE the vignette overlay (line 27):

```css
/* Compound hover shadow — compositor-safe pseudo-element (GLRY-02) */
/* Gated to hover-capable devices to prevent touch flicker (PERF-04) */
@media (hover: hover) and (pointer: fine) {
  .gallery-card {
    position: relative;
  }
  .gallery-card::after {
    content: '';
    position: absolute;
    inset: 0;
    border-radius: 2px;
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.7), 0 2px 8px rgba(0, 0, 0, 0.5);
    opacity: 0;
    transition: opacity 0.25s ease;
    pointer-events: none;
    z-index: 1;
  }
  .gallery-card:hover::after {
    opacity: 1;
  }
}
```

Key details:
- `@media (hover: hover) and (pointer: fine)` gates ALL hover visual effects to desktop-only (PERF-04). Touch devices get no shadow pseudo, no stuck hover states.
- `pointer-events: none` on the pseudo-element prevents it from blocking card clicks or lightbox opening.
- `z-index: 1` keeps shadow below the grain/vignette overlays (z-index 9998/9999) but above the card content for visual stacking.
- The `::after` pseudo-element opacity trick avoids direct `box-shadow` animation which triggers GPU repaints every frame. Only opacity transitions — compositor-thread only.
- Do NOT use `will-change: transform` on the gallery cards — Motion handles GPU layer promotion internally via whileInView.

3. The existing gradient overlay in GalleryGrid.tsx (the div with `opacity-0 group-hover:opacity-100`) already works correctly for both touch and desktop because opacity transition without gradient shift is fine on touch tap-dismiss. Leave it as-is — no changes needed to the inner overlay div.
