1. Inside the `<svg>` element (after the opening tag, before the first motion.path), add a `<defs>` block with the ink-bleed filter:

```tsx
<defs>
  <filter
    id="ink-bleed"
    x="-10%"
    y="-10%"
    width="120%"
    height="120%"
    colorInterpolationFilters="sRGB"
  >
    <feTurbulence
      type="fractalNoise"
      baseFrequency="0.04 0.08"
      numOctaves={3}
      seed={2}
      result="noise"
    />
    <feDisplacementMap
      in="SourceGraphic"
      in2="noise"
      scale={6}
      xChannelSelector="R"
      yChannelSelector="G"
      result="displaced"
    />
    <feGaussianBlur in="displaced" stdDeviation={1.2} />
  </filter>
</defs>
```

Key details for the filter:
- `x="-10%" y="-10%" width="120%" height="120%"` expands the filter region beyond the element bounding box — WITHOUT this, displaced/blurred pixels outside the original bounds are silently clipped, making stroke edges look cut with scissors.
- `feTurbulence` generates organic noise for edge variation. `baseFrequency="0.04 0.08"` gives anisotropic noise (more horizontal texture). `seed={2}` is arbitrary but deterministic.
- `feDisplacementMap scale={6}` displaces stroke edges by up to 6px using the noise — creates irregular, hand-drawn edge quality.
- `feGaussianBlur stdDeviation={1.2}` adds the feathered bleed effect on top of the displacement.
- Use JSX numeric attribute syntax (curly braces) for numOctaves, seed, scale, stdDeviation — these are numbers, not strings.

- The `<h1>` itself is NOT a motion element — it is a static container. Only the inner motion.span orchestrates the stagger.
- `inline-flex flex-wrap justify-center` ensures letters line up horizontally and wrap on small screens.
